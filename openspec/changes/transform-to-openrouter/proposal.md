## Why

当前 one-api-libra 是一个基础的 API 代理平台，缺乏现代化的用户体验和商业化功能。OpenRouter.ai 已经验证了统一 AI 模型 API 平台的市场需求，其核心特点包括：

- 300+ 模型，60+ 供应商的统一访问
- 美元余额系统，按使用量付费
- 智能路由和故障转移
- 现代化的模型市场 UI
- Provider 路由和负载均衡

将 one-api-libra 改造为 OpenRouter 风格的平台，可以：
1. 提供更好的用户体验
2. 支持商业化运营
3. 实现智能模型路由
4. 增强系统可靠性

## What Changes

### Phase 1: 核心后端改造
- 实现美元余额系统（已部分完成）
- 完善模型定价管理
- 实现智能渠道路由算法
- 添加渠道健康检查和故障转移

### Phase 2: API 兼容层
- 完善 OpenAI 兼容 API
- 实现 OpenRouter 风格的 API 端点
- 添加模型列表 API
- 实现供应商路由参数

### Phase 3: 前端改造
- 重新设计首页（模型市场风格）
- 实现模型列表/搜索/筛选页面
- 添加模型排行榜
- 改造额度管理页面
- 完善 API Key 管理界面
- 添加活动记录和统计页面

### Phase 4: 高级功能
- Google OAuth 登录
- 渠道优先级和权重配置
- 请求重试和降级策略
- 用量预警和通知

## Impact

- **后端代码**: controller/, model/, relay/, router/
- **前端代码**: web/src/
- **数据库**: 新增表和字段
